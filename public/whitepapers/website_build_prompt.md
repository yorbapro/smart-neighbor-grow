# Prompt for Website-Builder Agent: Publish BrightLaunchIQ Whitepaper Series

**Objective:**

Your task is to publish the 13-part "AI Receptionist & AI Voice Agent Authority Papers" series to the BrightLaunchIQ.com website. These whitepapers are executive-level assets and should be presented professionally within a dedicated "Case Studies" section of the site.

---

**Assets:**

All 13 whitepapers are complete, formatted in Markdown, and available in a public Google Drive folder. Each file is named according to its title.

*   **Primary Asset Folder:** [https://drive.google.com/open?id=1KPboA_jQnRRXCgviOvqxC5QpvAnfpj6N](https://drive.google.com/open?id=1KPboA_jQnRRXCgviOvqxC5QpvAnfpj6N)

---

**Implementation Plan:**

1.  **Create a Main "Case Studies" Landing Page:**
    *   Create a new, top-level page on brightlaunchiq.com titled "Case Studies & Whitepapers."
    *   This page should serve as a central hub for all 13 papers.
    *   Design this page to be visually appealing, professional, and easy to navigate.
    *   For each of the 13 whitepapers, create a summary card or list item on this landing page. Each card should include:
        *   The full title of the whitepaper.
        *   The complete "Executive Summary" from the corresponding whitepaper file.
        *   A clear "Read More" or "View Case Study" button that links to the dedicated page for that whitepaper.

2.  **Create Individual Whitepaper Pages:**
    *   For each of the 13 whitepapers, create a new, dedicated page on the website.
    *   The content for each page should be the full Markdown content from the corresponding file in the Google Drive folder.
    *   Ensure the Markdown is parsed correctly and rendered as clean, readable HTML, preserving all headings (H1, H2, H3), bold text, tables, and hyperlinked references.
    *   The page layout should be consistent with the existing design and branding of BrightLaunchIQ.com.

3.  **Navigation:**
    *   Add a link to the new "Case Studies & Whitepapers" landing page in the main navigation menu of the website to ensure it is easily discoverable.

---

**On-Page SEO & URL Structure:**

For each of the 13 individual whitepaper pages, implement the following SEO best practices:

*   **URL Slug:** Create a clean, keyword-rich URL slug based on the whitepaper title. For example:
    *   **Title:** `AI Receptionist for HVAC Companies`
    *   **URL:** `brightlaunchiq.com/case-studies/ai-receptionist-for-hvac-companies`
*   **Page Title (`<title>` tag):** Use the full whitepaper title.
    *   *Example:* `<title>AI Receptionist for HVAC Companies | BrightLaunchIQ Case Study</title>`
*   **Meta Description:** Use the first 155-160 characters of the "Executive Summary" as the meta description for each page.
*   **Headings:** The whitepaper title should be the main `<h1>` heading on the page. All other headings from the Markdown (`##`, `###`) should be rendered as `<h2>`, `<h3>`, etc.
*   **Links:** Ensure all hyperlinks in the "References" section of each whitepaper are rendered as clickable `<a>` tags with the correct `href` attribute.
